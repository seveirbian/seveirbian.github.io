有很多情况需要从终端访问墙外的网站，例如从谷歌系下载文件等等。这次我们介绍一下如何使用shadowsocks从终端“Fan Qiang”。我们假设已经有一个shadowsocks服务端，我们这里只需要配置客户端即可

<!-- more -->

## 事先准备

1. 一个shadowsocks服务端，配置信息为：
```
ip: AAA.AAA.AAA.AAA
port: BBBB
password:CCCC
```

2. git，安装方法：`sudo apt-get install git`


## 部署步骤

### 安装、配置和启动shadowsocks客户端

1. 安装python-pip（如果已安装则跳过）：`sudo apt-get install python-pip`

2. 安装shadowsocks：`sudo pip install shadowsocks`

3. 编写shadowsocks客户端配置文件，存储在喜欢的位置（这里假设在`~/`目录下，即`~/cfg.json`）:
```
{
    "server":"AAA.AAA.AAA.AAA", # 填入shadowsocks服务端配置信息中的ip地址
    "server_port":BBBB, # 填入shadowsocks服务端配置信息中的端口
    "local_port":DDDD, # 设置shadowsocks客户端在本地监听的端口
    "password":"CCCC", # 填入shadowsocks服务端配置信息中的密码
    "timeout":600,
    "method":"aes-256-cfb"
}
```

4. 启动shadowsocks客户端：`sslocal -c ~/cfg.json -d start`


### 安装和配置proxychains

1. 下载源码：`git clone https://github.com/haad/proxychains.git`

2. 编译安装：
```
./configure
make
sudo make install
```

3. 复制配置文件（src/proxychains.conf）到`~/.proxychains`目录，即`~/.proxychains/proxychains.conf`，并修改内容
```
[ProxyList]
# add proxy here ...
# meanwile
# defaults set to "tor"
#socks4  127.0.0.1 9050
socks5 127.0.0.1 DDDD #填入shadowsocks客户端本地监听端口
```

### 愉快翻墙

在任何需要翻墙的指令前加前缀`proxychains4`即可，例如：`proxychains4 curl www.google.com`

## Debug

1. 启动shadowsocks客户端时出现`undefined symbol: EVP_CIPHER_CTX_cleanup`错误：
```
原因：openssl高版本中，修改了EVP_CIPHER_CTX_cleanup为EVP_CIPHER_CTX_reset；而shadowsocks代码中未作修改
解决方案：
1. 打开/usr/local/lib/python2.7/dist-packages/shadowsocks/crypto/openssl.py文件
2. 搜索cleanup关键字并将其全部替换成reset
3. 保存文件并重启shadowsocks客户端即可
```
